Section 2: Visual Readability

Hierarchy, labels, hover, idle gaps, and minimap polish

2a — Hierarchy connector lines

Subtle vertical guide lines show which child rows belong to which parent, making the tree structure scannable at a glance.

cargo build 0.8s
clang-A 0.3s
clang-B 0.45s
cc1 0.38s

Vertical line + horizontal tick drawn in nodeRow() per depth level. Color inherits from parent node, at 35% opacity.

2b — Idle gap shading

Translucent hatched regions between a parent's start and its first child (or between siblings) reveal wait time — the wasted time you can't see today.

idle

Gaps ≥ 50ms get a hatched overlay. Color is amber. Tooltip on hover shows exact idle duration.

2c — Rich hover tooltip

Replace the plain .help() string with a floating card on hover showing timing context. Appears near the cursor, disappears on move-away.

Implemented with onHover + an overlay anchored to the hovered node. No click required.

2d — Minimap critical path overlay

Critical path nodes get a gold outline in the minimap (matching the timeline's gold border), so you can instantly see the hot path even when zoomed into a detail.

Gold = critical path

Requires passing criticalPathIDs to MinimapView (it currently only gets the timeline).
